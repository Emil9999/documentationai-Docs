<Update label="2025-01-15" description="v1.0 - Breaking change for pagination" tags={["Breaking Change"]}>
  ### Pagination now required

  - `GET /jobs` and `GET /candidates` now require `pagination=true` query parameter to receive full result sets
  - Without `pagination=true`, results are capped at the first 25 records
  - Update your integration to explicitly opt in to paginated responses

  <Callout kind="alert">
    Existing integrations that relied on implicit full-result behavior will silently truncate results. Verify your pagination handling.
  </Callout>
</Update>

<Update label="2024-09-15" description="v0.8 - Authentication upgrade" tags={["Authentication"]}>
  ### Bearer API Key authentication

  - New Bearer API Key authentication recommended over legacy JWT
  - Legacy JWT method continues to work but is being phased out
  - API keys can be generated and revoked from the [Kini Partner App](https://app.getkini.com/)
</Update>
