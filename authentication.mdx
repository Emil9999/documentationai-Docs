## Authentication Methods

The Backend API supports multiple authentication methods depending on the client and the endpoint being accessed.

### 1. Session Authentication
For web-based clients and internal users, the API uses standard Django session authentication.
- **Header**: `Cookie: sessionid=<session_id>`
- **Usage**: Primarily used by the internal admin panel and frontend dashboard.

### 2. Token / API Key Authentication
For programmatic access and external integrations, requests must include an API token.
- **Header**: `Authorization: Token <your_api_token>`
- **Usage**: Used by external services and background workers.

### 3. Magic Links
For passwordless, secure access to specific company or application resources, the API generates short-lived, token-based **Magic Links**.
- **Endpoint**: `/api/magic/<token>/`
- **Usage**: Allows candidates or company representatives to access specific pages without a full user account.

## Public Endpoints

Certain endpoints are public and do not require authentication. These are prefixed with `/api/public/` and are used by public-facing job boards and application forms.

- **Public Jobs**: `/api/public/jobs/`
- **Public Companies**: `/api/public/companies/`
- **Public Application Submission**: `/api/public/applications/`
