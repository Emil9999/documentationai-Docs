## System Requirements

Before you begin, ensure you have the following installed on your local machine:

- **Git**
- **Python 3.11**
- **PostgreSQL 15**
- **Google Cloud CLI**

## Local Setup

Follow these steps to get the service running locally:

<Steps>
  <Step title="Clone the Repository" icon="git-branch">
    Clone the `backend-api` repository to your local machine.
  </Step>

  <Step title="Setup Virtual Environment" icon="terminal">
    Create a Python virtual environment and install the required dependencies:
    ```bash
    python3 -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    ```
  </Step>

  <Step title="Install Pre-commit Hooks" icon="shield-check">
    Install pre-commit and configure it with git hooks:
    ```bash
    pip install pre-commit
    pre-commit install
    ```
  </Step>

  <Step title="Authenticate with Google Cloud" icon="key">
    Authenticate with the Google Cloud CLI to access cloud resources:
    ```bash
    gcloud auth application-default login
    ```
  </Step>

  <Step title="Run Database Migrations" icon="database">
    Run the migrations to set up your local SQLite database:
    ```bash
    python manage.py migrate
    ```
  </Step>

  <Step title="Start the Local Server" icon="play">
    Start the Django development server:
    ```bash
    python manage.py runserver
    ```
    The API will be available at `http://127.0.0.1:8000/`.
    You can access the interactive API documentation at `http://127.0.0.1:8000/docs/`.
  </Step>
</Steps>

## Connecting to the Dev Database (Optional)

If you need to connect to the development PostgreSQL database locally, you can use the Cloud SQL Proxy:

1. Install the proxy:
   ```bash
   brew install cloud-sql-proxy
   ```
2. Update your `.env` file:
   ```env
   ENVIRONMENT=localdev
   ```
3. Run the proxy:
   ```bash
   cloud-sql-proxy qonexon-production:europe-west3:dev-postgres
   ```
